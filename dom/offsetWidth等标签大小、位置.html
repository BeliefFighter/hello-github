document.body.clientWidth 和 document.documentElement.clientWidth 区别 (注意宽度都不包括滚动条)：
如果页面声明 DOCTYPE：document.body.clientWidth 是 body 中内容可见区域宽(只包括padding)，document.documentElement.clientWidth 是浏览器可视内容区域宽度
如果页面没声明 DOCTYPE：document.body.clientWidth 是浏览器可视内容区域宽度，document.documentElement.clientWidth 是 html 中内容可见区域宽(只包括padding)
以上 FF、chrome 都适应，IE 只适应声明 DOCTYPE 情况。
对于 IE 没声明 DOCTYPE：
IE8、IE9 中，没有设置<meta http-equiv="X-UA-Compatible" content="IE=8" />来保持 IE8/IE9 模式，浏览器模式会变为“杂项”，这时：
document.documentElement.clientWidth 始终为0，document.body.clientWidth 浏览器可视部分宽度
IE10、IE11 中没有特别的设置，document.documentElement.clientWidth == document.body.clientWidth == 浏览器可视部分宽度

document.documentElement.clientHeight 始终为0，document.body.clientHeight 浏览器可视部分高度

dom.offsetLeft;
与 offsetTop 类似
在Chrome、Opera、Safari、IE8、IE9和IE10中，工作正常，document.body.offsetLeft的值为0。
在FireFox中，offsetLeft的值为负的body元素的border-left-width。
在IE7中，offsetLeft指的是body元素对象的边框（不包括边框）到document左边的距离。

document.body.scrollWidth 和 document.documentElement.scrollWidth 区别 (注意宽度都不包括滚动条)：
如果页面声明 DOCTYPE：
FF、IE：document.body.scrollWidth --> body中内容宽度，document.documentElement.scrollWidth --> 浏览器中所有内容宽度
chrome：document.body.scrollWidth --> 浏览器中所有内容宽度，document.documentElement.scrollWidth --> html中内容宽度
如果页面没声明 DOCTYPE：
FF：document.body.scrollWidth --> 浏览器中所有内容宽度，document.documentElement.scrollWidth --> html中内容宽度
chrome：与页面声明 DOCTYPE 一样
IE10/11：document.body.scrollWidth == document.documentElement.scrollWidth == 浏览器中所有内容宽度
IE8/9：还是建议写上 DOCTYPE 吧

dom.scrollHeight;标签中所有内容的高度，如果没有出现竖直滚动条，那么 scrollHeight==clientHeight

dom.scrollTop; 获取滚动条向下滚动过的像素数，也就是内容部分上边已经隐藏的高度，等价于$("#testDiv1").scrollTop()
在 FF 里：如果设置 DOCTYPE，使用 document.documentElement.scrollTop，否则使用 document.body.scrollTop。
在 chrome 里：都使用 document.body.scrollTop。
在 IE 里：如果设置 DOCTYPE，使用 document.documentElement.scrollTop，不建议去掉 DOCTYPE。
有的浏览器使用window.pageYOffset，比如safari 、chrome，兼容写法：
var scrollTop = document.documentElement.scrollTop || window.pageYOffset || document.body.scrollTop;
